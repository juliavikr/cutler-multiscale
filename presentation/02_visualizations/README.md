# Visualizations

## Where they come from

Generated by `tools/visualize_pseudo_masks.py` on the cluster. Run:

```bash
sbatch slurm/run_visualize.sh
```

Output lands at `experiments/visualizations/baseline/` on the cluster (excluded from Git).
Transfer to your Mac with:

```bash
rsync -avz 3355142@slogin.hpc.unibocconi.it:~/cutler-multiscale/experiments/visualizations/ \
    presentation/02_visualizations/generated/
```

The script produces PNGs named `<image_id>_<n>masks.png` — 20 random images by default,
with colored mask overlays and bounding boxes.

## Which images are worth using

For slides and the report, prioritise images that tell a clear story:

| Category | What to look for | Use in |
|---|---|---|
| **Success cases** | Multiple distinct masks on clearly separated objects | Slide 8, report Fig. 2 |
| **Small-object wins** | A small mask that single-scale would have missed | Slide 8 headline figure |
| **Failure cases** | Over-segmentation, merged objects, background bleed | Slide 9 |
| **Scale comparison** | Same image, baseline vs multiscale side-by-side | Slide 8 / report Fig. 3 |

## Recommended figure count

- Report: 2–3 figures (1 pipeline diagram + 1–2 mask overlays)
- Slides: 3–4 images spread across slides 7–9
